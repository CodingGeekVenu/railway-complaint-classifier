# Railway Complaint Classification & Optimization

This project analyzes and classifies railway passenger complaints into predefined departments using Natural Language Processing (NLP). It includes scripts to train a baseline model, optimize it with hyperparameter tuning, and evaluate its performance.

## Project Structure

- **`Training/`**: Contains the raw training data, including complaint bodies and their corresponding labels.
- **`Testing/`**: Contains the raw testing data for evaluating the final model.
- **`PNR Mapping/`**: Includes JSON files for mapping PNR and zone data.
- **`baseline.py`**: Trains a baseline Logistic Regression model and saves the initial model and a confusion matrix.
- **`optimize.py`**: Runs a `GridSearchCV` to find the best hyperparameters for the classification pipeline, evaluates the optimized model, and saves the final model and its confusion matrix.
- **`optimized_railway_model.joblib`**: The final, optimized classification model.
- **`*.png`**: Confusion matrices generated by the scripts.

## How to Run

### 1. Baseline Model
To train the initial model, run:
```bash
python baseline.py
```
This will output a classification report and save `baseline_model.joblib` and `baseline_confusion_matrix.png`.

### 2. Hyperparameter Optimization
To run the optimization and find the best model, execute:
```bash
python optimize.py
```
This script will perform an exhaustive search, print the best parameters, evaluate the final model, and save `optimized_railway_model.joblib` and `optimized_confusion_matrix.png`.
